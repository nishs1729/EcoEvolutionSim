Your idea is actually very close to how **modern evolutionary theory models strategy evolution** (adaptive dynamics, quantitative genetics, phenotype landscapes). The key is to **separate three layers**:

1. **Trait space (genotype/phenotype parameters)**
2. **Behavioral rules generated from those traits**
3. **Fitness outcomes produced by interactions**

Strategies like “lekking” or “nuptial gifts” should **not exist as explicit states**. They should instead **emerge automatically from the trait vector**.

# 1. Step 1 — Define an Evolvable Trait Space

Each individual has a **continuous trait vector**

$$\mathbf{z} = (z_1, z_2, ..., z_n)$$

where each ($z_i$) represents an **evolvable biological trait**.

Example mutation process:

$$z_i' = z_i + \epsilon$$

where $\epsilon \sim N(0,\sigma^2)$.

This ensures **small mutations move individuals locally in phenotype space**.

---

# 2. Step 2 — Choose Fundamental Behavioral Axes

The trick is to define axes that are **biologically primitive** rather than strategy-specific. A useful decomposition for mating systems might look like this:

### Axis 1 — Signaling Investment

Energy spent producing signals. Examples:

* pheromones
* acoustic calls
* visual display

Trait:

```
signal_strength
```

High values → strong courtship display
Low values → silent strategies

---

### Axis 2 — Resource Investment in Mating

Energy allocated to **nutritional investment in females**.\
Trait:

```
nuptial_investment
```

Low → no gifts \
High → large spermatophores or prey gifts

---

### Axis 3 — Mate Search vs Mate Attraction

Movement strategy.\
Trait:

```
search_strategy
```

Range:

```
0 = sit and signal
1 = actively search for females
```

Intermediate values give mixed strategies.

---

### Axis 4 — Territorial Aggression

Probability of defending a spatial location.

Trait:

```
territoriality
```

High values produce:

* territory defense
* resource-based mating systems

Low values produce:

* scramble competition

---

### Axis 5 — Mate Guarding

Propensity to stay with female after mating.

Trait:

```
guarding_duration
```

---

### Axis 6 — Coercion / Force

Propensity to attempt mating without female choice.

Trait:

```
coercion_level
```

High values generate things like:

* forced copulation
* traumatic insemination

---

### Axis 7 — Female Choosiness

For female agents:

```
choosiness
```

Controls:

* probability of rejecting low quality males

---

### Axis 8 — Sperm Competition Investment

Male allocation to ejaculate size.

```
ejaculate_size
```

---

# 3. Step 3 — Behavioral Rules Derived From Traits

Strategies emerge from **decision functions**.

Example: male movement

```
if search_strategy > random():
    move_randomly()
else:
    remain_stationary_and_signal()
```

Example: female mate choice

```
accept_probability = sigmoid(
    male_signal_strength * female_choosiness
)
```

Example: gift production

```
gift_size = nuptial_investment * energy
```

---

# 4. Step 4 — Interaction Mechanics

Agents interact in space.

Possible processes:

### Mate encounter

```
if distance(male,female) < mating_radius:
    mating_attempt()
```

### Female acceptance

```
P_accept = f(
    male_signal,
    gift_size,
    female_choosiness
)
```

### Male coercion

```
P_force = coercion_level
```

---

# 5. Step 5 — Fitness Function

Fitness emerges from:

```
offspring_number × offspring_survival
```

Where:

```
offspring_number = egg_production + nuptial_gift_bonus
```

Costs include:

```
energy_cost_signal
energy_cost_gift
injury_cost_coercion
territory_cost
```

Thus fitness becomes:

```
fitness = reproduction - costs
```

---

# 6. Step 6 — Evolutionary Dynamics

Reproduction:

```
offspring_traits = parent_traits + mutation
```

Mutation:

$$z_i' = z_i + Normal(0, σ)$$


Selection acts automatically.

---

# 7. Step 7 — Strategies Emerge as Regions of Trait Space

After evolution, clusters of individuals appear.

Examples:

### Lekking strategy

Traits:

```
signal_strength = high
search_strategy = low
territoriality = low
```

Males gather in display sites.

---

### Nuptial gift strategy

Traits:

```
nuptial_investment = high
signal_strength = moderate
```

---

### Territorial strategy

Traits:

```
territoriality = high
signal_strength = moderate
```

---

### Scramble competition

Traits:

```
search_strategy = high
signal_strength = low
territoriality = low
```

---

### Coercive mating

Traits:

```
coercion_level = high
female_choosiness = high
```

# 8. Step 8 — Evolutionary Landscape

Fitness becomes a function of trait vector:

[
W(\mathbf{z})
]

Population evolves on this landscape.

Adaptive peaks correspond to **stable mating systems**.

---

# 9. Step 9 — Detecting Strategies Automatically

After simulation you can cluster individuals in trait space.

For example using PCA or clustering:

```
kmeans(traits)
```

Clusters correspond to **emergent mating strategies**.

# 10. Why This Works

This framework satisfies your requirement:

* strategies are **not discrete**
* evolution moves via **small mutations**
* strategies appear as **attractors in phenotype space**

This is conceptually similar to models used in:

* adaptive dynamics
* quantitative trait evolution
* evolutionary game theory


Often **6–10 trait dimensions are enough** to generate huge behavioral diversity.

The real complexity comes from:

```
trait interactions
+ spatial structure
+ frequency dependent selection
```

These produce multiple evolutionary equilibria.

Trait vector could influence:

```
mate choice
dispersal
reproductive compatibility
```

Then speciation can occur via:

```
assortative mating + spatial separation
```
